## AI Workflow

The pipeline runs **3 sequential Gemini calls**, each with a focused system prompt and a **strict JSON contract** so the output can be parsed and stored directly:

```
Product Image + Brand Brief (brand name, description, audience)
        │
        ▼
┌─────────────────────────────────────────────────┐
│  Step 1 — Vision Analysis                       │
│  Model: gemini-2.5-flash (multimodal)           │
│  Input: product image + brand context           │
│  Output: category, brand tone, audience         │
│          insights (age, lifestyle, pain points, │
│          motivations), platform priorities,     │
│          visual attributes, competitive angle   │
└─────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────┐
│  Step 2 — Content Engine                        │
│  Model: gemini-2.5-flash                        │
│  Input: Step 1 analysis JSON + brand context    │
│  Output: 3 Instagram captions (with tone),      │
│          3 ad units (Meta / Google / IG),       │
│          5 scroll-stopping hooks, 5 CTAs,       │
│          15 hashtags (primary/niche/trending)   │
└─────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────┐
│  Step 3 — Creative Director                     │
│  Model: gemini-2.5-flash                        │
│  Input: Step 1 + Step 2 JSON                    │
│  Output: 2 reel storyboards (shot list, music   │
│          vibe, duration, hook line),            │
│          2 carousel ideas (slide-by-slide),     │
│          2 campaign concepts (hero message,     │
│          channels, KPIs, duration)              │
└─────────────────────────────────────────────────┘
```

**Why this design?** Each step is a single-responsibility prompt with a narrow JSON schema, so the model stays focused and outputs are reliably parseable. Feeding Step 1's analysis into Steps 2 & 3 means the copy and creative are grounded in the same strategic reading of the product — no contradictions between the brand tone declared upstream and the captions written downstream.

## Setup (Local)

### Prerequisites
- Python 3.11+
- Node.js 18+
- A free Gemini API key → [aistudio.google.com/apikey](https://aistudio.google.com/apikey)

### 1. Backend

```bash
cd backend

# Create virtual environment
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # macOS/Linux

# Install dependencies
pip install -r requirements.txt

# Configure environment
copy .env.example .env         # Windows  (cp on macOS/Linux)
# Edit .env and paste your GEMINI_API_KEY
# Optional: GEMINI_MODEL=gemini-2.5-flash  (default)

# Start server
uvicorn main:app --reload --port 8000
```

- Health check: `http://localhost:8000/api/health`
- Swagger docs: `http://localhost:8000/docs`
- The SQLite DB and `uploads/` folder are created automatically on first run.

### 2. Frontend

```bash
cd frontend
npm install

# Optional — only if pointing at a non-local backend:
# copy .env.example .env  and set VITE_API_BASE_URL=https://your-backend-url

npm run dev
```

Open: `http://localhost:5173`

By default the frontend talks to `http://localhost:8000`. To point at a remote backend, set `VITE_API_BASE_URL` in `frontend/.env` (or in Vercel's env vars for production).

## Deployment

| Service | Platform | Notes |
|---|---|---|
| Frontend | **Vercel** | Auto-builds from `frontend/` on push. Env var `VITE_API_BASE_URL` points at the Render backend. |
| Backend | **Render** | Python web service. Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`. Env vars: `GEMINI_API_KEY`, optional `GEMINI_MODEL`. |

CORS on the backend is set to `allow_origins=["*"]` for portfolio/demo purposes — for a production multi-tenant deploy, lock it down to the Vercel domain.

## Features

### 1. Product Upload
- Drag-and-drop image upload (JPEG / PNG / WEBP, max 6 MB)
- Brand name, product description, target audience fields
- Live preview before submit

### 2. AI Product Analysis
- Product category classification
- Brand tone identification
- Audience demographics, pain points & motivations
- Platform priority recommendations (Instagram, TikTok, LinkedIn) with rationale & content-format suggestion
- Visual attributes & competitive angle

### 3. AI Content Generator
- 3 × Instagram captions (each with a tone label)
- 3 × Ad copy units (headline + body, per platform)
- 5 × Scroll-stopping hooks
- 5 × CTA suggestions
- 15 hashtags across 3 groups (Primary / Niche / Trending)
- **One-click copy** on every output

### 4. AI Creative Suggestions
- 2 × Reel storyboards (concept, shot list, music vibe, duration, hook line)
- 2 × Carousel ideas (slide-by-slide breakdown + CTA slide)
- 2 × Campaign concepts (hero message, channels, KPIs, duration)

### 5. Dashboard
- Project grid with per-step status indicators (Analysis / Content / Creative)
- Per-step re-run controls in the Workspace
- Delete projects
- Real-time progress pills during pipeline run

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/health` | Health check + Gemini key status |
| POST | `/api/projects` | Create project (multipart form: image + brand fields) |
| GET | `/api/projects` | List all projects |
| GET | `/api/projects/{id}` | Get single project (incl. analysis/content/creatives) |
| DELETE | `/api/projects/{id}` | Delete project |
| POST | `/api/projects/{id}/analyze` | Run Step 1 — Vision Analysis |
| POST | `/api/projects/{id}/content` | Run Step 2 — Content Engine |
| POST | `/api/projects/{id}/creatives` | Run Step 3 — Creative Director |

## Design

The UI follows an **editorial brutalist** aesthetic — Klein blue (`#002FA7`) as the only accent, sharp typographic hierarchy (`Outfit` for headings, `IBM Plex Sans` for body, `JetBrains Mono` for tags/labels), and a strict black-border grid system with zero rounded corners.

## Notes

- **Database** auto-creates on first run via `create_db_and_tables()` in `main.py` — no migrations to run.
- **`.gitignore`** excludes `venv/`, `.env`, `*.db`, `uploads/*`, `node_modules/`, and `dist/` — the repo stays clean and no secrets are committed.
- **Gemini model** is configurable via `GEMINI_MODEL` in `backend/.env` (default: `gemini-2.5-flash`).
- **Hot-reload caveat** — Uvicorn's `--reload` flag does not always pick up `.env` changes; if you update `GEMINI_API_KEY` mid-session, restart the server cleanly.
