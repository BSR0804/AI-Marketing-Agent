# MARKET/AGENT — AI Marketing Agent

> Drop a product photo. Get a complete launch kit — strategic analysis, ready-to-post copy, hashtags, and a creative storyboard — generated by a **multi-step Gemini agent** in under a minute.

**Live Demo**
- Frontend (Vercel): [ai-marketing-agent-swart.vercel.app](https://ai-marketing-agent-swart.vercel.app/)
- Backend (Render): [ai-marketing-agent-q0ne.onrender.com](https://ai-marketing-agent-q0ne.onrender.com/)

> The live frontend talks to the live backend. Render's free tier sleeps after ~15 min of inactivity, so the first request after a cold start can take 30–50 seconds — subsequent ones are instant.

## AI Workflow

The pipeline runs **3 sequential Gemini calls**, each with a focused system prompt and a strict JSON contract:

```
Product Image + Brand Brief
        │
        ▼
┌─────────────────────────────────────────────────┐
│  Step 1 — Vision Analysis                       │
│  Model: gemini-2.5-flash (multimodal)           │
│  Input: product image + brand name + brief      │
│  Output: category, tone, audience insights,     │
│          platform priorities, visual attributes │
└─────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────┐
│  Step 2 — Content Engine                        │
│  Model: gemini-2.5-flash                        │
│  Input: Step 1 analysis JSON                    │
│  Output: 3 Instagram captions, 3 ad units,      │
│          5 hooks, 5 CTAs, 15 hashtags            │
└─────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────┐
│  Step 3 — Creative Director                     │
│  Model: gemini-2.5-flash                        │
│  Input: Step 1 + Step 2 JSON                    │
│  Output: 2 reel storyboards, 2 carousel ideas,  │
│          2 full campaign concepts with KPIs      │
└─────────────────────────────────────────────────┘
```

Each step uses a **strict JSON contract** — the prompt demands only valid JSON output, which is parsed and stored in SQLite for the workspace view.

## Setup

### Prerequisites
- Python 3.11+
- Node.js 18+
- A free Gemini API key → [aistudio.google.com/apikey](https://aistudio.google.com/apikey)

### 1. Backend

```bash
cd backend

# Create virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS/Linux

# Install dependencies
pip install -r requirements.txt

# Configure environment
copy .env.example .env
# Edit .env and paste your GEMINI_API_KEY
# Optional: override the model with GEMINI_MODEL=gemini-2.5-flash (default) / gemini-2.0-flash-lite / etc.

# Start server
uvicorn main:app --reload --port 8000
```

API docs available at: `http://localhost:8000/docs`

### 2. Frontend

```bash
cd frontend

npm install
npm run dev
```

Open: `http://localhost:5173`

## Features

### 1. Product Upload
- Drag-and-drop image upload (JPEG / PNG / WEBP, max 6 MB)
- Brand name, product description, target audience fields

### 2. AI Product Analysis
- Product category classification
- Brand tone identification
- Audience demographics, pain points & motivations
- Platform priority recommendations (Instagram, TikTok, LinkedIn)
- Visual attributes & competitive angle

### 3. AI Content Generator
- 3 × Instagram captions (with tone label)
- 3 × Ad copy units (headline + body, per platform)
- 5 × Scroll-stopping hooks
- 5 × CTA suggestions
- 15 hashtags across 3 groups (Primary / Niche / Trending)
- One-click copy for every output

### 4. AI Creative Suggestions
- 2 × Reel storyboards (concept, shot list, music vibe, duration)
- 2 × Carousel ideas (slide-by-slide breakdown)
- 2 × Campaign concepts (hero message, channels, KPIs, duration)

### 5. Dashboard
- Project grid with status indicators (Analysis / Content / Creative)
- Per-step re-run controls in the Workspace
- Delete projects

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/health` | Health check + Gemini key status |
| POST | `/api/projects` | Create project (multipart form) |
| GET | `/api/projects` | List all projects |
| GET | `/api/projects/{id}` | Get single project |
| DELETE | `/api/projects/{id}` | Delete project |
| POST | `/api/projects/{id}/analyze` | Run Step 1 — Vision Analysis |
| POST | `/api/projects/{id}/content` | Run Step 2 — Content Engine |
| POST | `/api/projects/{id}/creatives` | Run Step 3 — Creative Director |

## Design

The UI is built around an **editorial brutalist** aesthetic — Klein blue (`#002FA7`), sharp typographic hierarchy (`Outfit` for headings, `IBM Plex Sans` for body, `JetBrains Mono` for code), and a strict black-border grid system with zero rounded corners.

## Notes

- SQLite database is auto-created on first run (`create_db_and_tables()` in `main.py`) — no manual migration step.
- The repo `.gitignore` excludes `venv/`, `.env`, `*.db`, `uploads/*`, and `node_modules/` — reviewers get a clean clone.
- The Gemini model is configurable via `GEMINI_MODEL` in `backend/.env` (default: `gemini-2.5-flash`).
